üî∏ 1. Inline CSS

Definition:

Inline CSS is used to apply a unique style directly to a single HTML element using the style attribute.

Deep Dive into Inline CSS:

Inline CSS works by directly injecting styles into HTML elements through the style attribute. It has the highest specificity in the CSS cascade, meaning it will override any other CSS rules targeting the same element. The specificity value for inline styles is 1000, compared to 100 for IDs and 10 for classes.

Important: Inline styles are parsed and applied immediately during HTML rendering, which can impact initial page load performance if overused.

Syntax:

<p style="color: blue; font-size: 16px;">This is an inline styled paragraph.</p>

Common Inline CSS Properties:

  • style="display: none;" - Hides an element
  • style="visibility: hidden;" - Makes element invisible but preserves space
  • style="position: absolute;" - Removes from normal document flow

Use Cases:

  • Quick styling
  • Dynamic styling via JavaScript
  • Email templates (due to limited CSS support)

Pros:

  • Quick and easy for small changes
  • High specificity (overrides internal and external styles)

Cons:

  • Not reusable
  • Clutters HTML
  • Difficult to maintain

üî∏ 2. Internal CSS

Definition:

Internal CSS is defined within a <style> tag in the <head> section of an HTML document.

Deep Dive into Internal CSS:

Internal CSS follows the standard CSS cascade rules and specificity calculations. It allows for the use of all CSS selectors, pseudo-classes, and media queries. The styles are parsed when the HTML document loads and remain in the browser's memory for that page only.

Internal CSS supports advanced CSS features like:

  • CSS Variables (Custom Properties)
  • @keyframes animations
  • @media queries for responsive design
  • CSS Grid and Flexbox layouts

Syntax:

<!DOCTYPE html>
<html>
<head>
  <style>
    p {
      color: green;
      font-size: 18px;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using internal CSS.</p>
</body>
</html>

Use Cases:

  • Single-page websites
  • Styling unique to that specific HTML file

Pros:

  • Keeps styles centralized within the document
  • Better than inline for reusability

Cons:

  • Cannot be reused across multiple HTML files
  • Adds to page size if used on many pages

üî∏ 3. External CSS

Definition:

External CSS is defined in a separate .css file and linked to the HTML document using the <link> tag.

Deep Dive into External CSS:

External CSS represents the most scalable approach to styling web applications. It implements the separation of concerns principle by completely separating structure (HTML) from presentation (CSS).

Key concepts:

  • Browser Caching: External CSS files can be cached by browsers, improving load times for returning visitors
  • CSS Modules: Multiple CSS files can be organized by functionality
  • Build Process: External CSS can be minified and optimized during deployment
  • CDN Distribution: CSS files can be served from Content Delivery Networks for better performance

Syntax (HTML):

<head>
  <link rel="stylesheet" href="styles.css">
</head>

Syntax (styles.css):

p {
  color: red;
  font-size: 20px;
}

Use Cases:

  • Multi-page websites
  • Large-scale projects

Pros:

  • Fully separates content (HTML) from presentation (CSS)
  • Reusable across multiple pages
  • Easier to maintain

Cons:

  • Requires an additional HTTP request (unless cached)
