CSS Transitions: Theory & Interactive Examples

What are CSS Transitions?

CSS transitions enable smooth, animated changes between property values, making web interfaces feel more dynamic and responsive. They are triggered by state changes (like :hover, :focus, or class changes) and are ideal for simple, one-step animations. Transitions are hardware-accelerated in modern browsers, making them efficient and performant.

How CSS Transitions Work

  • transition-property: The CSS property to animate (e.g., background-color, width).
  • transition-duration: How long the transition takes (e.g., 0.5s).
  • transition-timing-function: The speed curve of the transition (e.g., ease, linear, ease-in, cubic-bezier).
  • transition-delay: How long to wait before starting the transition (e.g., 0.2s).
  • transition: Shorthand for all the above.

Shorthand: transition: property duration timing-function delay;
Example: transition: background-color 0.5s ease 0.2s;

Animatable Properties

Not all CSS properties can be transitioned. Common animatable properties include color, background, opacity, width, height, transform, box-shadow, border-radius, and margin. Properties like display and position are not animatable.

Transition Timing Functions

  • ease: Starts slow, speeds up, then slows down (default)
  • linear: Constant speed
  • ease-in: Starts slow, then speeds up
  • ease-out: Starts fast, then slows down
  • ease-in-out: Starts and ends slow
  • cubic-bezier: Custom curve for advanced control
  • steps(): Jumps between steps for choppy effects

Try using cubic-bezier for unique, custom motion curves!

Best Practices & Notes

  • Only animatable properties can be transitioned (e.g., color, background, transform, opacity).
  • Use transition for simple, state-based animations (e.g., :hover, :focus).
  • For complex, multi-step animations, consider @keyframes and animation.
  • Keep transitions short and subtle for best user experience.
  • Test transitions on different devices and browsers for consistency.
  • Use will-change for performance optimization on heavy transitions.

Further Exploration

  • Combine transitions with :focus and :active for accessibility.
  • Experiment with cubic-bezier and steps() for custom effects.
  • Use transitions to provide visual feedback for UI elements (buttons, cards, menus, etc.).
