CSS Pseudo-Classes

Deep Dive: Theory, Usage, and Interactive Demos

What are CSS Pseudo-Classes?

Pseudo-classes are special keywords added to selectors that specify a special state of the selected elements. They allow you to style elements based on user interaction, position in the DOM, or other dynamic conditions, without adding extra classes or JavaScript.

  • User Interaction: :hover, :focus, :active
  • Structural: :first-child, :last-child, :nth-child(), :only-child, :empty
  • Form State: :checked, :disabled, :required, :valid, :invalid
  • Negation: :not()

Pseudo-classes are written after a selector, prefixed with a colon. Example: a:hover styles links when hovered.

Interactive Examples

:hover, :focus, :active

Style elements based on user interaction:

Hover, Focus, or Click Me 
.demo-btn:hover { background: #8e44ad; color: #fff; }
.demo-btn:focus { outline: 2px solid #f7971e; }
.demo-btn:active { background: #3498db; }

:first-child, :last-child, :only-child

Target elements based on their position in the parent:

  • First Child
  • Middle Child
  • Last Child
  • Only Child
.child-demo li:first-child { color: #8e44ad; font-weight: bold; }
.child-demo li:last-child { color: #3498db; }
.child-demo.single li:only-child { background: #ffd200; }

:nth-child(), :nth-of-type()

Style elements based on their index (1-based):

  1. Item 1
  2. Item 2
  3. Item 3
  4. Item 4
  5. Item 5
.nth-demo li:nth-child(odd) { background: #f0e6fa; }
.nth-demo li:nth-child(3) { color: #f7971e; font-weight: bold; }
.nth-demo li:nth-child(2n) { background: #e3f6fd; }

:empty

Target elements with no children (including text):

Not Empty
.empty-box:empty { border: 2px dashed #8e44ad; background: #f0e6fa; min-height: 2em; }

:not()

Exclude elements from a selection:

  • Apple
  • Banana
  • Cherry
.not-demo li:not(.exclude) { color: #3498db; font-weight: bold; }

Form State Pseudo-Classes

.form-demo input:required { border-color: #f7971e; }
.form-demo input:valid { border-color: #3498db; }
.form-demo input:invalid { border-color: #e74c3c; }
.form-demo input:checked + label { color: #8e44ad; font-weight: bold; }

Best Practices & Accessibility

  • Use :focus styles for keyboard accessibility.
  • Combine pseudo-classes for advanced effects (e.g., a:hover:active).
  • Test pseudo-class behavior across browsers and devices.
  • Use :not() to simplify complex selectors.
  • Prefer semantic HTML for better selector targeting.
