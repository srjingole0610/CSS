CSS Pseudo-Classes

Deep Dive: Theory, Usage, and Interactive Demos

What are CSS Pseudo-Classes?

Pseudo-classes are special keywords added to selectors that specify a special state of the selected elements. They allow you to style elements based on user interaction, position in the DOM, or other dynamic conditions, without adding extra classes or JavaScript.

  • User Interaction: :hover, :focus, :active, :focus-visible
  • Structural: :first-child, :last-child, :nth-child(), :only-child, :empty
  • Form State: :checked, :disabled, :required, :valid, :invalid
  • Negation & Logic: :not(), :is(), :where(), :has()

Pseudo-classes are written after a selector, prefixed with a colon. Example: a:hover styles links when hovered.

New & Advanced Pseudo-Classes

  • :is() — Matches any selector in a comma-separated list, making complex selectors easier to write and maintain. Example: :is(h1, h2, h3) { color: purple; }
  • :where() — Like :is() but always has zero specificity, useful for utility and reset styles. Example: :where(nav a) { text-decoration: underline; }
  • :has() — Parent selector: matches elements that contain a certain child or descendant. Example: article:has(img) { border: 2px solid #ffd200; }
  • :not() — Excludes elements matching a selector. Now supports complex selectors. Example: li:not(.active) { opacity: 0.7; }
  • :focus-visible — Styles elements only when focused via keyboard navigation (not mouse), improving accessibility. Example: button:focus-visible { outline: 2px solid #3498db; }

:is(), :where()

:is() and :where() simplify writing selectors for multiple elements. :where() always has zero specificity, so it won't override more specific rules.

:not() (Advanced)

:not() can now accept complex selectors, not just simple classes or elements. For example, style all list items except those with a specific class or attribute:

  • Banana

:focus-visible

:focus-visible only applies focus styles when the user is navigating with a keyboard (Tab), not with a mouse click. This improves accessibility and avoids unwanted outlines.

.focus-visible-demo-btn:focus-visible { outline: 3px solid #3498db; background: #e3f6fd; }

Best Practices & Accessibility

  • Use :focus styles for keyboard accessibility.
  • Combine pseudo-classes for advanced effects (e.g., a:hover:active).
  • Test pseudo-class behavior across browsers and devices.
  • Use :not() to simplify complex selectors.
  • Prefer semantic HTML for better selector targeting.
