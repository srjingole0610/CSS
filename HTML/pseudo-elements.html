CSS Pseudo-elements

Theory & Deep Explanation

Pseudo-elements in CSS are powerful tools that let you style specific parts of an element or insert content before or after it, without extra HTML. They enable advanced effects, custom highlights, and creative UI patterns, all while keeping your markup clean and semantic.

  • ::before / ::after: Insert generated content before/after the element's content. Great for icons, badges, or decorative elements.
  • ::first-letter: Style the first letter of a block element. Used for drop caps or magazine-style intros.
  • ::first-line: Style the first line of a block element. Useful for headlines or emphasis.
  • ::selection: Style the portion of text selected by the user. Enhances accessibility and branding.
  • ::placeholder: Style placeholder text in form fields. Improves form UX and branding.
  • ::marker: Style the bullet or number of list items. Allows for custom list styles.
  • ::backdrop: Style the background of modal elements (e.g., dialogs, popups).
  • ::file-selector-button: Style the button of file input fields for custom file upload UIs.
  • ::cue: Style WebVTT cues in media elements (advanced, for subtitles/captions).
  • ::part(): Style shadow DOM parts (advanced, for web components).

How Pseudo-elements Work

  • Pseudo-elements are not real DOM elements—they are rendered by the browser and cannot be selected with JavaScript.
  • They can be animated and transitioned for interactive effects.
  • They inherit styles from their parent unless overridden.
  • Some pseudo-elements (like ::before and ::after) require a content property to display.

Advanced Use Cases

  • Custom checkboxes, toggles, and switches using ::before and ::after.
  • Animated highlights, tooltips, and badges.
  • Accessible focus indicators and custom selection colors.
  • Decorative list markers and progress bars.
  • Styling form placeholders and file upload buttons for a consistent look.

Accessibility & Best Practices

  • Use pseudo-elements for decoration, not for essential content (screen readers may ignore them).
  • Always provide sufficient color contrast for ::selection and ::placeholder.
  • Test on different browsers for compatibility, especially with advanced pseudo-elements.

Code Examples

/* Animated badge with ::after */
.badge::after {
  content: '●';
  color: #e74c3c;
  margin-left: 8px;
  font-size: 1.2em;
  animation: pulse 1.2s infinite alternate;
}

/* Custom list marker */
ul.custom-list li::marker {
  color: #3498db;
  font-size: 1.2em;
}

/* Progress bar using ::before */
.progress-bar {
  position: relative;
  background: #eee;
  border-radius: 8px;
  height: 18px;
  width: 180px;
  margin: 10px 0;
}
.progress-bar::before {
  content: '';
  display: block;
  background: linear-gradient(90deg, #3498db 60%, #8e44ad 100%);
  height: 100%;
  border-radius: 8px 0 0 8px;
  width: 60%;
  transition: width 0.5s;
}

/* Custom placeholder and selection */
input::placeholder {
  color: #8e44ad;
  font-style: italic;
  opacity: 0.8;
}
input:focus::placeholder {
  color: #3498db;
  opacity: 1;
}
input::selection {
  background: #ffd200;
  color: #222;
}
