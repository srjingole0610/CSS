CSS Animations: Theory & Interactive Examples

What are CSS Animations?

CSS animations allow you to animate the values of CSS properties over time using @keyframes. Unlike transitions, which animate between two states, animations can define multiple steps and run automatically, loop, or alternate direction.

Animation Properties

  • animation-name: The name of the @keyframes to use.
  • animation-duration: How long one cycle of the animation takes (e.g., 2s).
  • animation-timing-function: The speed curve of the animation (e.g., ease, linear, steps, cubic-bezier).
  • animation-delay: How long to wait before starting the animation.
  • animation-iteration-count: How many times the animation repeats (1, infinite, or a number).
  • animation-direction: Whether the animation runs normal, reverse, alternate, or alternate-reverse.
  • animation-fill-mode: How styles are applied before/after animation (none, forwards, backwards, both).
  • animation-play-state: Whether the animation is running or paused.
  • animation: Shorthand for all the above.

Shorthand: animation: name duration timing-function delay iteration-count direction fill-mode;
Example: animation: bounce 1s ease-in-out 0.2s infinite alternate;

@keyframes

Define the animation steps using @keyframes:

@keyframes bounce {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-40px); }
  100% { transform: translateY(0); }
}

How CSS Animations Work

CSS animations use @keyframes to define multiple steps of an animation, allowing for complex, multi-stage effects. Animations can run automatically, loop, alternate, or be triggered by user interaction. Unlike transitions, which only animate between two states, animations can have as many steps as you want.

Animation Lifecycle

  1. Start State: The element's initial style.
  2. Keyframes: Define the intermediate and end states.
  3. Animation Properties: Control timing, repetition, direction, and more.
  4. End State: Determined by animation-fill-mode and keyframes.

Common Use Cases

  • Loading spinners and progress bars
  • Attention-grabbing UI feedback (e.g., shake, bounce, pulse)
  • Animated icons and illustrations
  • Background and color effects
  • Real-world simulations (e.g., pendulum, waves, clocks)

Interactive Examples

1. Basic Animation

Bounce!
@keyframes bounce {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-40px); }
  100% { transform: translateY(0); }
}
.anim-bounce {
  animation: bounce 1s ease-in-out infinite;
}

2. Animation Delay

Wait for it...
@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.anim-delay {
  animation: fadeIn 1.2s ease 1s forwards;
}

3. Iteration Count

3x Loop
@keyframes colorLoop {
  0%   { background: #3498db; }
  50%  { background: #8e44ad; }
  100% { background: #3498db; }
}
.anim-iterate {
  animation: colorLoop 1.2s linear 0s 3;
}

4. Animation Direction

Alternate
@keyframes slideX {
  from { transform: translateX(0); }
  to   { transform: translateX(60px); }
}
.anim-direction {
  animation: slideX 1.1s cubic-bezier(.4,1.6,.4,1) infinite alternate;
}

5. Fill Mode

Stay!
@keyframes grow {
  from { width: 120px; }
  to   { width: 220px; }
}
.anim-fill {
  animation: grow 1.2s ease-in 0s 1 forwards;
}

6. Multiple Animations

Combo
@keyframes spin {
  to { transform: rotate(360deg); }
}
@keyframes pulse {
  0%, 100% { box-shadow: 0 0 0 0 #8e44ad44; }
  50% { box-shadow: 0 0 16px 8px #8e44ad44; }
}
.anim-multi {
  animation: spin 1.2s linear infinite, pulse 1.2s ease-in-out infinite;
}

7. Image Animation

@keyframes imgPop {
  0%   { transform: scale(1) rotate(0deg); }
  50%  { transform: scale(1.13) rotate(-4deg); }
  100% { transform: scale(1) rotate(0deg); }
}
.img-anim-demo {
  animation: imgPop 1.4s cubic-bezier(.4,1.6,.4,1) infinite;
}

8. Pendulum Animation (Real-World Example)

@keyframes swing {
  0%   { transform: rotate(-30deg); }
  50%  { transform: rotate(30deg); }
  100% { transform: rotate(-30deg); }
}
.pendulum-arm {
  width: 6px;
  height: 90px;
  background: #8e44ad;
  margin: 0 auto;
  border-radius: 3px;
  transform-origin: top center;
  animation: swing 2s cubic-bezier(.4,1.6,.4,1) infinite;
}
.pendulum-bob {
  width: 32px;
  height: 32px;
  background: #ffd200;
  border-radius: 50%;
  margin: 0 auto;
  margin-top: 80px;
  box-shadow: 0 4px 16px #8e44ad44;
  border: 3px solid #8e44ad;
}

9. Loader Dots

@keyframes dotFlare {
  0%, 80%, 100% { opacity: 0.3; }
  40% { opacity: 1; }
}
.dot {
  display: inline-block;
  width: 14px;
  height: 14px;
  margin: 0 4px;
  background: #3498db;
  border-radius: 50%;
  animation: dotFlare 1.2s infinite both;
}
.dot2 { animation-delay: 0.2s; }
.dot3 { animation-delay: 0.4s; }

Best Practices & Notes

  • Use animation for multi-step, automatic, or looping effects.
  • Keep animations short and meaningful for best user experience.
  • Combine with transition and transform for advanced UI effects.
  • Test animations on different devices and browsers for consistency.
  • Use will-change for performance optimization on heavy animations.
  • Respect user preferences for reduced motion (see @media (prefers-reduced-motion)).

Further Exploration

  • Try animation-play-state for pause/resume effects.
  • Experiment with steps() and cubic-bezier() for custom timing.
  • Use animations for loaders, notifications, feedback, and more.
